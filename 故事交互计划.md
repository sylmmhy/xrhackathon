# Interactive Storytelling Demo Plan: "Dog & Bone"

## 1. Background & Goal

### Current State
- **Project**: sensai-webxr-worldmodels — a WebXR app using IWSDK (ECS framework) + SparkJS (Gaussian splat renderer) + Three.js
- **Existing capabilities**: distance grabbing, locomotion, hand tracking, dynamic GLB model loading, Meshy image-to-3D integration (via Yume backend)
- **Physics**: available in IWSDK (Havok-based) but currently disabled

### Vision
Build an "Alice in Wonderland" style interactive storytelling experience:
- User draws a dog → dog appears as 3D model
- User draws a bone → bone appears
- User grabs bone and brings it near the dog → dog runs over and eats the bone
- Dog grows huge → breaks through the house

### Scope of This Plan
**Single pre-scripted story demo** ("Dog & Bone") to validate the full pipeline. All 3D assets are pre-generated by Meshy AI. Users follow a guided narrative with interaction triggers.

### APIs/Technologies Selected
| Technology | Purpose |
|-----------|---------|
| **Meshy Image-to-3D API** | Pre-generate 3D models (dog, bone, house) |
| **Meshy Animation API** | Auto-rig dog model + assign animations (idle, run, eat) |
| **IWSDK Physics (Havok)** | Collision & debris effect for house destruction scene |
| **Three.js AnimationMixer** | Play GLB-embedded animations at runtime |
| **Claude API** | Future: AI-driven story logic (not in this phase) |

## 3. Story Script Format

The story is defined as a JSON file with this structure:

### Core Types
```typescript
interface StoryScript {
  id: string;
  title: string;
  assets: Record<string, AssetDef>;  // assetId -> asset definition
  scenes: SceneDef[];                // sequential scenes
}

interface AssetDef {
  glbUrl: string;                    // path relative to public/
  animations?: string[];             // animation clip names in the GLB
  defaultScale?: number;
}

interface SceneDef {
  id: string;
  name: string;
  onEnter: ActionDef[];              // execute immediately on scene entry
  triggers: TriggerDef[];            // conditional triggers
  onExit?: ActionDef[];              // cleanup on scene exit
}

interface TriggerDef {
  id: string;
  type: "proximity" | "grab" | "timer" | "animation_complete";
  params: Record<string, any>;       // varies by trigger type
  actions: ActionDef[];              // execute when trigger fires
  once: boolean;                     // fire only once?
  advanceTo?: string;                // transition to scene id
}

// Proximity trigger params:
interface ProximityParams {
  entityA: string;   // asset id
  entityB: string;   // asset id
  distance: number;  // meters
}

// Timer trigger params:
interface TimerParams {
  delayMs: number;
}

// Animation complete trigger params:
interface AnimCompleteParams {
  entityId: string;
  clipName: string;
}

type ActionType = "spawn" | "destroy" | "animate" | "scale"
               | "move_to" | "ui_prompt" | "physics_debris" | "set_grabbable";

interface ActionDef {
  type: ActionType;
  params: Record<string, any>;
}
```

## 4. Demo Story Flow (5 Scenes)

| Scene | Name | Trigger | Actions |
|-------|------|---------|---------|
| 1 | empty_room | Enter scene -> 3s timer | Show "Draw something!" prompt -> Spawn dog at [0, 0, -2], play idle animation |
| 2 | draw_bone | Enter scene -> 3s timer | Show "Draw something for the dog!" -> Spawn bone at [1, 1, -2], make it grabbable |
| 3 | feed_dog | Bone within 0.5m of dog | Dog plays "run" animation + moves toward bone -> plays "eat" animation |
| 4 | dog_grows | "eat" animation completes | Scale dog from 1x to 3x over 2 seconds, play idle |
| 5 | break_house | Enter scene -> 1s timer | Spawn house around dog -> dog runs toward wall -> physics debris explosion + camera shake |

### Visual Timeline
```
Scene 1          Scene 2          Scene 3          Scene 4          Scene 5
[Dog appears] -> [Bone appears] -> [Feed dog]   -> [Dog grows]  -> [Breaks house]
  idle anim        grabbable       proximity        scale 3x       physics debris
  3s delay         3s delay        user action      auto           auto
```

## 5. Key Component Details

### StoryEngineSystem (Core Orchestrator)
- Follows IWSDK ECS pattern: `createSystem()` with query for `StoryController` component
- **`update(delta)`** runs every frame:
  1. Check all triggers in current scene
  2. For proximity: call `ProximitySystem.getDistance(entityA, entityB)`
  3. For grab: check if entity has IWSDK's `Pressed` tag component (auto-added when grabbed)
  4. For timer: track elapsed time since scene entry
  5. For animation_complete: query `AnimationManager.isPlaying()`
  6. When trigger fires: execute actions sequentially, optionally advance to next scene
- Maintains `Map<string, Entity>` to track spawned entities by story asset id

### ProximitySystem (Lightweight Distance Detection)
- Custom ECS System using `object3D.getWorldPosition()` to compute distances
- Used for Scenes 1-4 (no physics overhead)
- `getDistance(entityA, entityB): number` — called by StoryEngineSystem

### AnimationManager (Animation Playback)
- Wraps `THREE.AnimationMixer` — one mixer per animated entity
- When GLTFLoader loads a GLB, captures `gltf.animations` (AnimationClip[])
- API: `play(entity, clipName)`, `stop(entity)`, `crossFadeTo(entity, clipName, duration)`, `onComplete(entity, clipName): Promise<void>`
- All mixers updated in `StoryEngineSystem.update(delta)` via `mixer.update(delta)`

### StoryAssetManager (Asset Loading)
- Extends pattern from existing `objectLoader.ts` (`spawnGLBFromUrl`)
- `loadStoryAsset(world, assetDef, position?)` — loads GLB, registers animations, returns entity
- `preloadAssets(world, assets)` — preloads all GLBs at story startup to avoid mid-scene hitches
- Caches loaded GLTF objects
- Applies `Interactable` + `DistanceGrabbable` only when requested via `set_grabbable` action

### Physics Strategy (Two-Tier)
- **Scenes 1-4**: No physics. ProximitySystem handles distance checks. Dog movement is scripted tween/lerp.
- **Scene 5 only**: Physics enabled.
  - `World.create()` set to `physics: true` (system initializes but has zero overhead until entities get physics components)
  - Spawn `house-debris.glb` pieces as separate entities with `PhysicsBody` (Dynamic) + `PhysicsShape` (ConvexHull)
  - Apply outward impulse forces to simulate explosion

### Gaussian Splat Environment Interaction
- Splats are purely visual with no collision geometry
- "Dog breaks through wall" effect is faked:
  1. Spawn GLB wall/house pieces overlaid at splat wall position
  2. Apply physics to scatter the GLB pieces
  3. Optional: fade/distort the splat region
  4. Camera shake effect for impact feel

## 6. Action Executors (Detail)

All actions implement a common interface:
```typescript
interface ActionExecutor {
  execute(context: StoryContext, params: Record<string, any>): Promise<void>;
}

interface StoryContext {
  world: World;
  assetManager: StoryAssetManager;
  animationManager: AnimationManager;
  spawnedEntities: Map<string, Entity>;
  currentScene: SceneDef;
}
```

| Action | What It Does |
|--------|-------------|
| **spawn** | Calls `assetManager.loadStoryAsset()`, stores entity in spawnedEntities, optional scale-in animation |
| **animate** | Calls `animationManager.play(entity, clipName)`. For "dog runs to bone": combines move_to tween + run clip |
| **scale** | Smoothly interpolates `entity.object3D.scale` over duration using lerp in update loop |
| **move_to** | Tweens entity position toward target (straight line, no pathfinding) |
| **destroy** | Removes entity from world, disposes geometry/materials |
| **physics_debris** | Spawns fractured GLB pieces with PhysicsBody + PhysicsShape, applies impulse forces |
| **ui_prompt** | Shows/hides HTML overlay with story text (same approach as existing uploadUI.ts) |
| **set_grabbable** | Adds `Interactable` + `DistanceGrabbable` components to entity |

## 7. Asset Preparation Pipeline

Run in parallel with code development:

1. **Meshy Image-to-3D API**: Generate `dog.glb` and `bone.glb` from reference images
2. **Meshy Animation API** (Pro tier):
   - Auto-rig dog model (humanoid or quadruped)
   - Select animations: `idle`, `run`, `eat` from 500+ preset library
   - Export as single GLB with embedded animation clips
3. **House debris**: Either:
   - Generate house via Meshy, fracture in Blender (Cell Fracture addon)
   - Or manually create simple box fragments as `house-debris.glb`
4. **Inspect GLB animations**: Use Three.js Editor or `gltf-transform inspect` to verify clip names match story.json

## 11. Known Challenges & Mitigations

| Challenge | Mitigation |
|-----------|-----------|
| `physics: false` in World.create() may prevent PhysicsSystem from initializing | Change to `physics: true` upfront; zero overhead when no entities have physics components |
| Meshy auto-rig animation quality may vary | Pre-test dog model, manually select best animation presets |
| GLB animation clip names from Meshy are generic | Inspect with gltf-transform, map names in story.json |
| "Dog runs to bone" is straight-line (no pathfinding) | Acceptable for demo scope; user places bone in open area |
| Splat environment has no collision geometry | Fake destruction by overlaying GLB pieces at splat wall position |
